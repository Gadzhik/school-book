# Фича: Создание книги из фотографий страниц

> Аддитивное дополнение к `chitalka_promt_i_tehplan.md`. Не заменяет ТЗ. Новый модуль, встраивается в существующую архитектуру.

## Суть

Пользователь фотографирует страницы по одной (или загружает готовые фото), приложение их обрабатывает, распознаёт текст и **собирает в одну книгу**, которая попадает в библиотеку и открывается в обычной читалке.

По сути это «сканер документов → книга». Большая часть кирпичей уже есть в основном ТЗ: OCR (Tesseract), выходные форматы EPUB/CBZ/PDF, единая модель документа, библиотека.

## Куда встраивается в архитектуру

- **Новый пакет:** `packages/book-scanner` — захват, обрезка, улучшение, управление страницами, оркестрация OCR, сборка.
- **Переиспользует:** `packages/converters` (OCR через Tesseract), вывод в единую модель документа и библиотеку из `packages/core`.
- **Новый адаптер:** `camera` / `capture` в `packages/adapters` (рядом со storage/files/tts) — платформенный доступ к камере.
- **UI:** экраны сканера в `packages/ui`.
- **Зависимость по фазам:** требует OCR из **Фазы 2**. Если OCR ещё не готов — сначала сделать сборку из изображений (CBZ/PDF без текста), а распознавание подключить позже.

## Конвейер обработки

| Этап | Что делает | Технология |
|---|---|---|
| 1. Захват | Фото с камеры или загрузка готовых изображений | getUserMedia / нативная камера / `<input capture>` |
| 2. Поиск страницы | Авто-определение краёв листа в кадре | **jscanify** или **OpenCV.js** (corner detection) |
| 3. Коррекция перспективы | Выправление перекоса (фото под углом) | OpenCV.js (warpPerspective) |
| 4. Улучшение | Контраст, удаление теней, бинаризация («Ч/Б документ») | canvas-фильтры / OpenCV |
| 5. Управление страницами | Лента: порядок, перетаскивание, переснять, удалить, вставить | UI |
| 6. OCR | Распознавание текста по страницам (rus+eng) | **Tesseract.js** в Web Worker |
| 7. Сборка | Объединение в книгу нужного формата | см. ниже |
| 8. Сохранение | Обложка, название/автор → в библиотеку | `packages/core` |

## Форматы на выходе (дать выбор пользователю)

| Формат | Что внутри | Когда выбирать | Плюсы / минусы |
|---|---|---|---|
| **Searchable PDF** *(по умолчанию)* | Изображение страницы + невидимый слой OCR-текста | Учебники, точная вёрстка, формулы, таблицы | Точная вёрстка + поиск; но не reflow, TTS по слою хуже |
| **Reflowable EPUB** | Чистый текст из OCR | Художка, главы текстом | Работают TTS, масштаб шрифта, дислексия-шрифт, «Мои слова»; но видны ошибки OCR |
| **CBZ** | Только изображения, без OCR | Самый быстрый путь, комиксы, плохое качество фото | Мгновенно; не ищется, не озвучивается |

**Рекомендация:** по умолчанию — **searchable PDF** (Tesseract.js умеет выдавать PDF с текстовым слоем напрямую). Дополнительно предложить «Преобразовать в текстовую книгу (EPUB)» с **шагом проверки/правки OCR** перед сборкой — тогда раскрываются все школьные фичи читалки.

## Платформенные нюансы (где болит)

- **Камера:** на вебе проще всего `<input type="file" accept="image/*" capture="environment">` — вызывает родной интерфейс камеры. На мобилках (Tauri mobile / Capacitor) — **нативный плагин камеры**: лучше качество и работа с памятью.
- **Память — главный риск.** Фото высокого разрешения × много страниц = переполнение ОЗУ на слабых устройствах. Обязательно:
  - обрабатывать **по одной странице**, не держать все в RAM;
  - даунскейл до ~2000px по длинной стороне перед сохранением;
  - промежуточные изображения хранить в **OPFS / нативной ФС**, а не в памяти;
  - освобождать canvas после каждой страницы.
- **OpenCV.js тяжёлый** (~8 МБ WASM) — грузить **лениво**, только при открытии сканера, не в основной бандл.
- **OCR медленный** — в Web Worker, с прогрессом по страницам, не блокируя UI.

## UX-поток (для школьника)

1. «Создать книгу из фото» → камера.
2. Снял страницу → **авто-обрезка по краям** (углы можно поправить пальцем) → **авто-улучшение** → «Добавить ещё» / «Готово».
3. Лента страниц: порядок, перетаскивание, переснять, удалить.
4. Обложка (первая страница или своя), название и автор.
5. Выбор формата сборки → OCR с прогресс-баром → книга в библиотеке.

Всё «магическое» (обрезка, улучшение) — в одно нажатие, ручная правка — как запасной вариант.

## Обработка «грязных» входов

Реальные фото от детей будут плохими — заложить это сразу:
- **Размытие** → детектировать, предупредить, предложить переснять.
- **Палец / тень в кадре** → поправка краёв вручную.
- **Блик/глянец** на мелованной бумаге → фильтр снижения бликов.
- **Изгиб у корешка**, перекос, разное освещение между страницами → коррекция перспективы + по-страничное улучшение.

## Приватность и бонус

- Всё локально: камера, OCR, сборка. Фотографии **не уходят в облако** — в духе общего приватного подхода.
- **Тай-ин с локальной LLM (Ollama):** после OCR прогнать текст через локальную модель для исправления опечаток распознавания и расстановки абзацев. Опционально, офлайн.
